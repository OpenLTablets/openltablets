Web Studio Overview

The Webstudio allows users to browse OpenL Project structure, select Project Components, edit Project Components directly in Excel; in addition it can Run and Trace the Executable Components. The results of Run are displayed in the browser and are available to inspect.

This document demonstrates and briefly explains how to use OpenL Tablets Web Studio. Starting with version 4.0.1 Web Studio provides a better interface for developers to work with OpenL Tablets projects than the Eclipse plugin. It provides better error display, advanced features like visualizing unit test results, tracing and explanation.

We still use Eclipse to do some of work, in paticular - organize OpenL projects, compile java code, run Ant scripts, provide an interface to CVS etc. For all the activities related to the editing of the project's files, fixing errors, running, testing, benchmarking etc. we highly recommend using OpenL Web Studio.

General View

Web Studio General View consists of the three frames: Top Menu, Project Explorer and Project Component View

Top Menu

Top Menu Provides controls over Studio's most general tasks. It consists of the following items:

  • Project Selector combobox - allows to select any available OpenL Project in Web Studio workspace. If used with Eclipse IDE, Web Studio workspace coincides with Eclipse workspace. In this case combobox should contain all OpenL Projects. It is possible to use Web Studio in standalone mode(no Eclipse IDE). In standalone mode the user will not have an option to recompile java classes of the project (we use Eclipse for this) but all other Studio's features will be available.
  • Studio Control Buttons:
    • Business Mode - provides "business user friendly" view of the project. It should be noted that Business Mode hides some of the project parts and table parts from the user. For more details on Business Mode and how to control it's appearance look here
    • Developer Mode - provides full view of the project. It is more suitable for a project developer/administartor.
    • Search - link to the project Search Page. OpenL will automatically index all the documents in project's folder and provide an instant access to them via familiar internet-like search interface. In addition to the text search Web Studio now provides Advanced Search that provides table-specific search capabilities.
    • Refresh - use this button if you a) changed any of the project files b) recompiled any of the project's java classes using Eclipse IDE. Studio will reload the project, validate it and display the new version.
    • Help - displays Studio online Help

Project Explorer

The Project Explorer provides familiar tree-like view that allows users to browse project components. Usually one project component correspond to the one table in Excel. In the Business Mode components are organized by categories. In the Developer Mode the components are organized by component's type or by component's location.

    Project menu

On top of the Project Tree there is a Project Menu. It contains the following buttons:

  • Run All Tests - this button runs and displays all the unit tests (if any) contained in the project. It is recommended to run all the unit tests after you made any changes.
  • Refresh - use this button if you a) changed any of the project files b) recompiled any of the project's java classes using Eclipse IDE. Studio will reload the project, validate it and display the new version.
  • Expand All - expands all tree branches
  • Collapse All - collapses all tree branches to the first level

    Project Component Types and Icons

  • Decision Table, Decision Table With Error, Decision Table With Unit Test(s)
  • Data Table, Data Table With Error
  • Datatype Table, Datatype Table With Error
  • Method Table, Method Table With Error
  • Unit Test Table, Unit Test Table With Error
  • Run Method Table, Run Method Table With Error
  • Environment Table, Environment Table With Error
  • Other - tables that do not correspond any of the defined above Component Types. They are considered as comments or text, but no validation checks are generally provided for these tables

To display Project Component in Project Component View click on it's link.

    Project Folders

  • Closed Folder, Closed Folder With Errors Inside
  • Open Folder, Open Folder With Errors Inside
  • Excel Workbook, Excel Workbook With Errors Inside
  • Excel Worksheet, Excel Worksheet With Errors Inside

To open or close any of the project folders click on it's icon. You can also use Expand All and Collapse All icons in Project Menu to open or close folders.

Project Component View

This view is used to display project components (mostly tables), results of unit tests and runs and search results

The Table View consist of the following components:

  • The link to the original document in format: <Document Name>:<Worksheet Name>:<Table Name>
  • The set of functional buttons with the following functionality:
    • Run - runs Unit Tests and Runnable Methods (optional)
    • Trace - produces a viewable trace of the Run (optional)
    • Test - runs all the Unit Tests (if any) applicable to this component (optional)
    • Business Mode - provides a "business user friendly" view of the component, usually by hiding it's technical details.
    • Developer Mode - provides a full view of the component.
  • Error View - if the component contains any errors they will be displayed next (optional)
  • Component View - usually a table, displayed as close as possible to it's original Excel look
  • Run List - if the component has any Named Runs associated with it they will be displayed as a group of hyperlinks (optional)
