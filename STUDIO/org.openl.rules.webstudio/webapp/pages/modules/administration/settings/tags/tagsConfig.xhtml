Tag types and tag values

To add a tag type, type tag type name in “New Tag Type” field and hit Enter/Tab. As a result, tag type will be added and fields for filling tag values will appear.

Add tag values by typing tag values names and hitting Enter.

If tag type is nullable, this tag type can be skipped when create a new project. Otherwise, it's mandatory.

If tag type is extensible, any user can create a new tag values for this type. Otherwise, tag values are configured by admin only.

All created tag types and values are saved automatically.

Tags from project name

Project name template is used to extract tags from project name.

Tag types are wrapped with '%' symbol. '?' means any symbol. '*' means any text with any length.

For example, if we have template %Domain%-%LOB%-*, then for project Policy-L&A-rules tags will be Policy for tag type Domain and L&A for LOB.

Each template should be in a new line.

The order of templates is important: first template has highest priority, last template has lowest priority.
